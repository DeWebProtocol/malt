# MIP Template

Copy this file when drafting a new MALT Improvement Proposal. Replace the
front-matter values and remove guidance text before moving the MIP to Review.

```markdown
---
mip: 1010
title: Short Title
description: One sentence summary.
author: MALT maintainers
status: Draft
type: Standards Track
category: Core
created: 2026-05-25
requires: none
replaces: none
---

## Abstract

Summarize the proposal in one short paragraph.

## Motivation

Explain the problem and why it matters now.

## Specification

Describe the proposed behavior or decision boundary precisely enough for
review. Link to `docs/spec/` or protocol schema files for durable
field tables, wire formats, and artifact definitions.

## Rationale

Explain alternatives and why this proposal chooses its approach.

## Backwards Compatibility

Describe compatibility impact, migrations, and legacy artifact handling.

## Security Considerations

Describe correctness, verification, trust, or misuse concerns.

## Implementation Plan

State whether implementation is approved. Draft MIPs should not contain active
implementation steps; accepted MIPs should point to the linked issue, PR plan,
or repository planning note.

## History

- YYYY-MM-DD: Created.
```
